# Contribuer

Cette documentation est volontairement simple : que du Markdown, lu directement sur GitHub.

## Conventions Markdown

- Titres en `#`, `##`, `###`. Jamais soulignés.
- Code blocks toujours avec un langage : ```` ```python ````, ```` ```text ````, ```` ```json ````.
- Tableaux pour les comparaisons.
- Liens internes en relatif.
- Pas de tirets cadratin (`—`). Utiliser `:`, `,`, `;`, des parenthèses ou un retour à la ligne.

## Pull requests

- Une PR = un changement focalisé.
- Titre court, en français ou en anglais.
- Description : ce qui change, et pourquoi.
